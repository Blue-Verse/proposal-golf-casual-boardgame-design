<프로젝트 진행 제안>

■ 프로젝트 분석
골프장 결제 서비스 앱 내에 골퍼들이 라운딩 중 간단히 즐길 수 있는 캐주얼 보드게임(라운딩 순서 정하기, 경품 추천 이벤트 게임 등)을 추가하기 위한 상세 기획 및 UI/UX 디자인 프로젝트로 이해하였습니다. 초기 아이디어 단계에서 요구사항 정의, 기능 정의, 화면 설계, 정책 설계를 거쳐 Figma 디자인 산출물까지 완성하는 것이 목표입니다.

■ 작업 일정

[Phase 1] Day 1–4 (리서치 · 아이디어 구체화)
- 유사 캐주얼 보드게임 벤치마킹 조사
- 게임 컨셉 정의 및 메커니즘 설계
- 산출물: 벤치마킹 리포트, 게임 컨셉 문서

[Phase 2] Day 5–8 (기능 정의 · 정책 설계)
- 라운딩 순서 정하기, 경품 추천 이벤트 게임 기능 요구사항 명세
- 게임 정책 설계 (보상 체계, 참여 조건, 결과 처리)
- 산출물: 요구사항 정의서, 기능 명세서, 게임 정책서

[Phase 3] Day 9–12 (화면 설계)
- 전체 화면 와이어프레임 (게임 로비, 진행, 결과, 경품 등)
- 화면 플로우 다이어그램 및 인터랙션 정의
- 산출물: 와이어프레임 (Figma), 화면 플로우

[Phase 4] Day 13–18 (UI/UX 디자인)
- 디자인 시스템 정의 (컬러, 타이포, 아이콘)
- 전체 화면 UI 디자인 및 인터랙티브 프로토타입
- 산출물: Figma 디자인, 프로토타입, 디자인 시스템

[Phase 5] Day 19–20 (검수 · 수정 · 납품)
- 클라이언트 피드백 반영 및 최종 산출물 정리
- 산출물: 최종 기획서, Figma 원본, 디자인 가이드

■ 마일스톤 및 산출물
- M1 (Day 4): 게임 컨셉 확정
- M2 (Day 8): 기능 명세 완료 (요구사항 정의서, 기능 명세서)
- M3 (Day 12): 와이어프레임 완료
- M4 (Day 18): UI 디자인 및 프로토타입 완료
- M5 (Day 20): 최종 납품 (상세 기획서 + Figma 디자인 원본)

■ 미팅 시 협의 필요 사항
- 기존 골프 앱의 디자인 시스템(컬러, 타이포, 컴포넌트) 공유 여부
- 게임 종류 및 범위 확정 (라운딩 순서 정하기, 경품 이벤트 외 추가 게임 여부)
- 경품 정책 상세 (경품 종류, 당첨 확률, 수령 방법 등)
- 개발팀과의 핸드오프 방식 (Figma 직접 접근 또는 에셋 전달)
- 기존 앱 내 게임 진입점 (탭, 배너, 플로팅 버튼 등) 위치 협의
